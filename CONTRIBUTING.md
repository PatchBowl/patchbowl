# Contributing to Patchbowl

Thanks for being here. Patchbowl is small, early, and shaped heavily by the people who show up first — that includes you.

## The fastest way to help

**Add an MCP server to the catalog.** It's one YAML file, no code, ~10 minutes start to finish. See [`catalog/README.md`](catalog/README.md) for the schema and a walkthrough.

## Other ways to help

- **Try Patchbowl and break it.** File an issue with what you tried and what happened.
- **Improve docs.** If something in the README or docs/ tripped you up, a PR fixing it is enormously valuable.
- **Pick up a `good-first-issue`.** We label these weekly. Comment on the issue before you start so we don't double up.
- **Bigger features.** Open an issue first to discuss the approach before writing code. Saves us both time.

## How we work

- **Issues before code.** For anything bigger than a typo fix, open an issue first.
- **PRs stay small.** One thing per PR. Easier to review, faster to merge.
- **Discussions in Discord, decisions in GitHub.** Real-time chat is for thinking out loud; the issue tracker is the record.
- **Be kind.** See [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
